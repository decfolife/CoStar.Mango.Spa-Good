# Introduction
---

Welcome to the Technical Documentation for the CREM Design System. 

This documentation serves several objectives:

1. **Integrate the Design System specifications**

Incorporate the design system specifications, such as Figma Tokens, Prototypes, and Designs, into the storybook stories within the shared library and how to use them.

2. **Streamline collaboration and evolve our design system**

Facilitate effective collaboration and support the ongoing improvement of our design system by providing clear guidelines for developers and designers to work together.

3. **Establish a decision-making process**

Implement a structured approach to determine whether a component should be included in the shared library or created as a one-off within micro-components or apps.

4. **Provide comprehensive code guidelines**

Furnish detailed information on code guidelines, encompassing best practices and frontend architecture principles, to promote code that is scalable, maintainable, and readable.
