# Single Properties
---

The main mixin for **single properties is `costar()`**. It takes two parameters: the CSS property name 
and the mixin parameter. The mixin parameter comes from the SCSS maps available at 
 `./common/src/styles/abstract/_variables.scss`.

Use the tables in here to find the mixin parameter for the component you want to style. 

# Usage
---

Utilize the `costar()` mixin to ensure full compatibility across different web browsers. 
The mixin will take care of both the CSS4 variable and the compiled variable value. 
Don't use the CSS4 (E.g. --primary-light) or thee SCSS (Eg. $primary) variables directly. 

E.g.
``` SCSS
button {
  @include costar(font-family, body);
  @include costar(border-radius, small);
  @include costar(font-size, normal);
  @include costar(transition, fast);
}
```

This approach enables convenient theming and manipulation of CSS4 variables directly in the browser, 
without relying on JavaScript. This can be particularly beneficial for enhancing accessibility and other 
customizations.

The SCSS mixin will compile into the CSS4 variable and the compiled value. E.g.:

``` SCSS
// SCSS
@include costar(background-color, gray07);

// Compiled CSS
background-color: #FBFBFB; // For legacy support on older browsers
background-color: var(--costar-color-gray07, #FBFBFB);
```

If the `costar()` mixin is not enough use the same CSS structure provided by the costar() mixin. 
However, avoid using SASS variables directly, as it would defeat the purpose of CSS4 variables 
(e.g., --costar-color-primary).

E.g.

``` SCSS
// Bad
.my-class {
  color: $primary;
}
// Good
.my-class {
  color: map-get($color, color, primary);
  color: var(--costar-color-primary);
}

```

### Breakpoints
Media queries or breakpoints facilitated through the 
[Include Media](https://eduardoboucas.github.io/include-media/documentation/) library 
and uses a different mixin: `@include media(...)`.

> Include-media is a Sass library for writing CSS media queries in an easy and maintainable way, using a natural and simplistic syntax.
> It provides Flexible declaration of breakpoints, On-the-fly breakpoints, Smart support for media types, Support for different units, etc.

The arguments are taken from the `_variables.scss` file and are used as follows:

**Available queries**: `phone`, `tablet`, `desktop`, `desktop-xl`, `desktop-xxl`, `landscape`, and `portrait`.

**Operators or Media Expressions:**: `>`, `<`, `>=`, and `<=`.

**Examples**:

  <tr>
    <th>Parameter 1</th>
    <th>Parameter 2</th>
    <th>Mixin</th>
    <th>Description</th>
  </tr>
  {/* phone & tablet */}
  <tr>
    <td rowspan="10">phone</td>
    <td>tablet</td>
    <td>
      <code>@include media("&gt;=phone", "&lt;=tablet") &#123; ... &#125; </code>
    </td>
    <td>Targets phones and tablets</td>
  </tr>
  {/* phone & tablet */}
  <tr>
    <td>desktop</td>
    <td>
      <code>@include media("&gt;=phone", "&lt;=desktop") &#123; ... &#125; </code>
    </td>
