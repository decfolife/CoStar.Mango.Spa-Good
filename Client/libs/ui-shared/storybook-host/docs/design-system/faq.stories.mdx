## What is Atomic Design (Atoms, Molecules, Organisms, etc.)?
Atomic Design is a methodology for designing and structuring user interfaces (UIs) in a modular and 
systematic way. It breaks down UI components into smaller, reusable building blocks called atoms, 
which are the basic elements like buttons, inputs, or labels. Atoms can be combined to form more complex 
components called molecules, which are small UI groups that have a specific functionality. Molecules, in 
turn, can be further combined to create larger and more comprehensive components called organisms, which 
are complete sections of a UI. This hierarchy continues with templates, which bring together different 
organisms to create complete pages, and pages, which represent the final screens seen by users. Atomic 
Design helps in creating consistent and scalable UI systems, making it easier for developers to build 
and maintain UI components.

*Our design system and shared library are based on Atomic Design principles.*

## What is the shared library?
Is the collection of components, following CoStar code best practices. It is designed to support micro-components 
and shared-components. It offers reusable functionality across multiple projects or applications within the organization. 
The shared library aims to promote code consistency, reduce duplication of effort, and improve efficiency in the software 
development process.

## What are Design Tokens and why they're important?
Design tokens are a set of predefined variables or constants that encapsulate design-related values, 
such as colors, typography, spacing, and other visual attributes. They serve as a single source of truth 
for design-related properties and front-end efforts, ensuring consistency across different platforms, 
devices, and applications.

The design tokens are defined first on Figma and then translated into the common library.

## What is a Design System (Figma)?

A Design System in Figma refers to a centralized collection of design assets, components, guidelines, 
and documentation that helps maintain consistency and efficiency in the design process. Figma is a popular 
cloud-based design tool that facilitates collaboration and enables designers to create, iterate, and share 
designs with ease.

By utilizing a Design System in Figma, design teams can work more efficiently, ensure consistency, 
and streamline collaboration. It promotes reusability, reduces duplication of efforts, and allows for rapid 
iteration and prototyping. Design Systems in Figma contribute to a more streamlined and cohesive design 
workflow, ultimately leading to better user experiences.

If you need access to Figma please contact the Product or UX team.

## What is a Production Design System (Storybook)?
A Production Design System in Storybook refers to a comprehensive set of design assets, components, 
guidelines, and documentation that are specifically geared towards facilitating the development and 
production of user interfaces.
