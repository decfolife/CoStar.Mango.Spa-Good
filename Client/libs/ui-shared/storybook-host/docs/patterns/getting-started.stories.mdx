# Patterns
Patterns refer to established and recurring solutions for common design challenges or interactions within 
a user interface. Patterns are predefined approaches or templates that provide a consistent and familiar 
experience to users across different screens or applications.

Some examples of patterns include: Loading pages, Forms, Navigations, Cards, Login Pages, Search, Modals, 
Tables, Etc.

Patterns help designers and developers maintain consistency, improve usability, and enhance  the overall 
user experience.
